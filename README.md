# 📡 内网二维码数据传输工具

一个面向物理隔离网络的离线数据摆渡工具。发送端把文本或文件压缩、分片并编码成二维码序列；接收端通过屏幕扫描、截图粘贴或图片拖拽自动识别、去重、拼合并还原数据。

## ✨ 特性

| 特性 | 说明 |
|------|------|
| 🔒 完全离线 | 发送端是单个 `export.html`，接收端只依赖本地 Flask 服务 |
| 📦 智能压缩 | 使用 pako zlib 自动压缩文本/代码，压缩无收益时自动回退原始数据 |
| 🧩 分片自愈 | 支持乱序扫描、重复去重、缺片提示，补扫缺片即可继续拼合 |
| 🎥 屏幕扫描 | 接收端通过 WebRTC 屏幕捕获 + jsQR 自动识别轮播二维码 |
| 📥 粘贴拖拽 | 支持 `Ctrl+V` 粘贴截图，也支持批量拖拽二维码图片 |
| 🖥️ 全屏轮播 | 发送端可全屏自动播放分片二维码，支持键盘调速和手动切换 |
| 🎨 简洁界面 | 发送端和接收端统一玻璃质感暗色 UI，突出当前任务与下一步操作 |

## 🧭 工作流

```text
发送端 export.html
  文本/文件 → zlib 压缩 → base64 编码 → 分片 → 二维码序列/全屏轮播

二维码画面
  ├─ 屏幕扫描 → jsQR 解码
  └─ 粘贴/拖拽图片 → jsQR 解码

接收端 Flask 页面
  分片去重 → 缺片提示 → base64 拼合 → pako 解压 → 文本复制/文件下载
```

## 🚀 快速开始

### 发送端

发送端只需要一个文件：

```text
export.html
```

双击打开即可使用；QRCode.js 和 pako 已内联，不需要安装依赖，也不需要联网。

### 接收端

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
启动网页.bat
```

启动后浏览器会打开本地页面，服务只监听 `127.0.0.1:5000`。

## 📖 使用方法

### 1. 生成二维码

1. 打开 `export.html`
2. 选择 **文本** 或 **文件**
3. 输入文本，或拖拽/选择文件
4. 按需调整“单个二维码容量”（默认 `400` 字符）
5. 点击 **生成二维码序列**
6. 可点击 **全屏轮播** 让接收端自动扫描

全屏轮播快捷键：

| 按键 | 功能 |
|------|------|
| `←` / `→` | 上一张 / 下一张 |
| `↑` / `↓` | 加速 / 减速 |
| `Space` | 播放 / 暂停 |
| `Esc` | 退出 |

### 2. 接收分片

接收端支持三种入口，可以混合使用：

- **屏幕扫描**：点击 **开启屏幕扫描**，选择发送端窗口或整个屏幕
- **粘贴截图**：截图二维码后，在接收页面按 `Ctrl+V`
- **拖拽图片**：把二维码截图批量拖入接收页面

接收端会自动识别分片任务、更新进度、标出缺失分片，并在全部集齐后停止屏幕扫描。

### 3. 还原数据

- **文本任务**：全部集齐后点击 **一键复制**
- **文件任务**：全部集齐后点击 **下载文件**
- **缺片任务**：复制缺片编号，到发送端跳转对应分片后补扫

## 🧩 分片协议

```text
文本未压缩：[CHUNK:1|TOTAL:5|HASH:a1b2c3d4]base64data
文本已压缩：[CHUNK:1|TOTAL:5|HASH:a1b2c3d4|COMP:1]base64data
文件已压缩：[CHUNK:1|TOTAL:5|HASH:a1b2c3d4|FILE:文件名Base64|COMP:1]base64data
```

- `CHUNK`：当前分片序号
- `TOTAL`：总分片数
- `HASH`：任务 ID，用于避免不同任务分片混入
- `FILE`：文件名的 Base64 编码，仅文件任务存在
- `COMP:1`：表示载荷经过 zlib 压缩

## 📊 性能参考

| 数据类型 | 压缩表现 | 10KB 预计分片 | 0.8s/片预计时间 |
|---------|----------|---------------|-----------------|
| 源代码 / 配置 | 压缩明显 | 约 9-12 片 | 约 7-10 秒 |
| Markdown / 纯文本 | 压缩明显 | 约 10-13 片 | 约 8-11 秒 |
| JSON | 压缩明显 | 约 10-14 片 | 约 8-12 秒 |
| ZIP / JPEG / MP4 | 通常回退原始传输 | 约 30+ 片 | 约 25 秒以上 |

> 适合传输代码、配置、密钥、脚本、小型文档等轻量数据；大文件建议先压缩或拆分后再传输。

## 📁 项目结构

```text
IntranetDataTransferTool/
├── export.html          # 发送端：单 HTML 离线页面，内联 QRCode.js + pako
├── app.py               # 接收端：Flask 本地服务
├── templates/
│   └── index.html       # 接收端页面：屏幕扫描 + 粘贴/拖拽 + 分片还原
├── static/
│   ├── jsqr-1.4.0.js    # 二维码解码库
│   └── pako.min.js      # zlib 解压库
├── requirements.txt     # Python 依赖
└── 启动网页.bat          # Windows 一键启动脚本
```

## ⚠️ 注意事项

- 接收端必须通过本地 Flask 页面打开，直接打开模板文件无法加载静态资源
- 浏览器首次开启屏幕扫描时会请求屏幕共享权限
- 如果识别不稳定，降低轮播速度到 `1.5s` 左右，或改用截图粘贴/拖拽
- 已压缩媒体文件通常不会再压缩，分片数量会更多
- 接收端默认只监听 `127.0.0.1`，不会对外暴露端口
