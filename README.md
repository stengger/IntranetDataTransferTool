# 📡 内网二维码数据传输工具

**专为物理隔离内网、保密网闸、无联网环境设计的全离线数据摆渡工具。**

将任意文本或文件压缩、分片，编码为二维码序列在屏幕上展示；接收端通过三条独立通道（截屏/天眼视频/粘贴拖拽）自动识别、去重拼合，实现跨物理隔离的单向数据传输。

---

## ✨ 核心特性

| 特性 | 说明 |
|------|------|
| 🔒 **完全离线** | 发送端零依赖，接收端仅需本地 Python，无任何公网请求 |
| 📦 **zlib 智能压缩** | 发送前自动 pako zlib 压缩，文本/代码类数据分片数减少 60-80% |
| 🧩 **自愈分片协议** | 乱序扫描、重复扫描均可自动去重；接收端直观指出缺漏片，补扫即愈 |
| 🎥 **天眼全自动监控** | WebRTC 屏幕捕流 + jsQR 实时解码，分片集齐后自动停止 |
| ⌨️ **F8 截屏通道** | 后端 pyzbar 高精度识别，支持 800+ 字节高密度 QR 码 |
| 🖥️ **全屏轮播大厅** | SVG 倒计时环，默认 0.8s 换片，键盘 ↑↓ 实时调速 |
| 📁 **文件传输** | 任意格式文件 base64 编码后分片传输，接收端一键无损还原下载 |

---

## 🏗️ 架构概览

```
发送端 export.html（纯离线单 HTML，双击即用）
  原始数据 → pako zlib 压缩 → base64 编码 → 按块分片
  → 带协议头的 QR 码序列 → 全屏轮播大厅展示

          ┌─── F8 截屏（通道 A）→ pyzbar 后端解码
屏幕展示 ──┼─── 天眼视频流（通道 B）→ jsQR 前端解码
          └─── 粘贴/拖拽图片（通道 C）→ jsQR 前端解码

接收端 index.html（Flask 本地服务，含智能拼图主控台）
  base64 分片拼合 → pako inflate 解压 → 原始数据/文件还原
```

### 分片协议头格式

```
文本模式（无压缩）：[CHUNK:1|TOTAL:5|HASH:a1b2c3d4]base64data
文本模式（已压缩）：[CHUNK:1|TOTAL:5|HASH:a1b2c3d4|COMP:1]base64data
文件模式（已压缩）：[CHUNK:1|TOTAL:5|HASH:a1b2c3d4|FILE:文件名Base64|COMP:1]base64data
```

> `COMP:1` 标志完全向下兼容——接收端旧版无此字段时自动走原始解码逻辑。

## 🛠️ 环境准备

### 发送端（内网机器）

**零依赖**，直接双击 `export.html` 用浏览器打开即可。  
所有依赖库（QRCode.js、pako.js）均已内联，无需联网。

### 接收端（本地/外网机器）

```bash
# 1. 安装 Python 依赖（清华镜像加速）
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 2. 双击启动（自动请求管理员权限以挂载 F8 全局热键）
启动网页.bat
```

> **为什么需要管理员权限**：F8 全局键盘钩子需要提权才能在任意前台窗口生效。
> BAT 文件会自动申请 UAC 提权、切换 UTF-8 代码页并打开浏览器。

**依赖项目**：

| 包 | 用途 |
|----|------|
| `flask` | 本地 Web 服务 + SSE 推流 |
| `keyboard` | F8 全局热键监听 |
| `Pillow` | 全屏截图（ImageGrab） |
| `pyzbar` | 后端 ZBar 二维码解码（高密度识别） |

## 📖 操作指南

### 第一步：发送端生成二维码

1. 双击打开 `export.html`
2. 选择模式：
   - **📝 文本模式**：粘贴代码、配置、密钥等文本内容
   - **📁 文件模式**：拖拽任意格式文件到投递舱（推荐文本类；视频/zip 等已压缩格式无压缩增益）
3. 调整单块容量滑块（默认 400 字节，最大 600 字节）
4. 点击 **【✨ 生成分块二维码序列】**
5. 点击 **【🖥️ 进入极速全屏轮播放映大厅】** 进入全屏自动轮播

**轮播大厅快捷键**：

| 按键 | 功能 |
|------|------|
| `←` / `→` | 上一张 / 下一张 |
| `↑` / `↓` | 加速 / 减速（步进 100ms） |
| `Space` | 播放 / 暂停 |
| `Esc` | 退出放映 |

---

### 第二步：接收端捕获数据

三条通道任意组合使用，数据流自动去重合并：

#### 🚀 通道 A — F8 截屏（最稳定，推荐）

1. 运行 `启动网页.bat` 启动接收端
2. 将发送端二维码显示在屏幕上
3. 每看到一张新的二维码，按 **`F8`** 触发后台截屏
4. 捕获成功时播放滴声并点亮对应插槽方格

#### 🎥 通道 B — 天眼全自动监控（解放双手）

1. 点击接收端右舱 **【🎥 开启屏幕全自动监控】**
2. 系统弹窗中选择共享「发送端窗口」或「整个屏幕」
3. 让发送端进入全屏轮播，天眼自动高频捕捉、去重入库
4. **分片全部集齐后天眼自动关闭**，释放屏幕录制权限

#### 📥 通道 C — 粘贴 / 拖拽（跨设备备用）

- **`Ctrl+V`**：直接粘贴截图工具截取的二维码图片
- **拖拽**：将批量二维码截图文件拖入蓝色投递区，自动排序解码

---

### 第三步：还原下载

**文本传输完成**：预览区实时显示已拼合内容，全部集齐后点击 **【📋 一键复制】**

**文件传输完成**：
- 插槽矩阵全部点亮为翡翠绿
- 点击 **【📥 一键无损还原下载】**，系统自动 pako 解压，以原始文件名保存到本地

**补扫漏片**：若插槽矩阵中有红色方格（如第 5、12 格），回到发送端用搜索跳转栏定位到对应分片展示，F8 或天眼补扫即可瞬间自愈。

## 📊 传输性能参考

| 数据类型 | 压缩率 | 10KB 数据预计分片数 | 传输时间（0.8s/片） |
|---------|--------|-------------------|------------------|
| Python/JS 源代码 | ~70% | 约 9 片 | ~7 秒 |
| JSON / 配置文件 | ~65% | 约 10 片 | ~8 秒 |
| 纯文本 / Markdown | ~65% | 约 10 片 | ~8 秒 |
| ZIP / JPEG / MP4 | 0%（自动回退） | 约 33 片 | ~26 秒 |

> 理论峰值吞吐：约 **1.4 KB/s** 原始数据（700 字节/片，0.5s/片）。
> 系统适合传输代码、配置、密钥等文本类数据，不适合传输大型二进制文件（视频等）。

## ⚠️ 注意事项

- **视频/音频/ZIP 等已压缩格式**：pako 压缩无增益，系统会自动检测并回退为原始字节传输，不影响功能，但分片数较多
- **天眼识别失败**：建议降低轮播速度至 1.5s 以上，或切换为 F8 截屏通道（pyzbar 识别率更高）
- **大文件传输**：超过 50KB 的文件建议先压缩为 zip 再传输（但 zip 本身不可再压缩），或分批传输
- **网络隔离要求**：接收端 Flask 仅监听 `127.0.0.1:5000`，不对外暴露端口
