# 📡 全通用内网二维码数据传输工具 - 项目使用说明书

本工具是一款专为**物理隔离内网、保密网闸、无网线环境**而设计的通用型、全离线数据传输工具。系统通过前端高级 Base64 分片切片技术，将大段文本或任何格式的文件转换为可自愈的二维码序列；接收端整合为 **【三通道大融合主控台】**，依靠高响应看门狗与像素自愈引擎，自动执行乱序收集、除重和物理重组，实现双手完全解放的高速无损传输。

---

## 🚀 核心架构与黑科技

### 1. 向下兼容的自愈分片协议
系统摒弃了低效的人工死板顺序扫描，采用带全局任务元数据的分片协议头：
* **文本模式**：`[CHUNK:当前分片|TOTAL:总分片|HASH:任务唯一ID]Base64数据`
* **文件模式**：`[CHUNK:当前分片|TOTAL:总分片|HASH:任务唯一ID|FILE:文件名Base64]Base64数据`

> [!NOTE]
> **去乱序与智能自愈机制**：接收端在内存桶中自动排重并物理重组。无论您是乱序扫描还是中途重复扫描，系统均能零阻碍识别；若中途因翻页过快产生“漏扫”，接收端将直观指出缺漏的方格，补扫漏片后即可瞬间还原。

### 2. 发送端「全屏定时幻灯片自动轮播放映大厅」
* **双手彻底解放**：生成二维码序列后，点击 **【🖥️ 进入极速全屏轮播放映大厅】**，即可享受全自动大图循环轮播，免除人工频繁按键或滚屏。
* **高精细倒计时环**：右上角部署了精美的 SVG 圆形倒计时环，每 `50ms` 优雅递减更新，让传输进度一目了然。
* **调速与多媒体控制**：可在 `200ms - 3000ms` 之间高频调速，自带播放、暂停、上一张、下一张多媒体控制栏。

### 3. 接收端「科幻双舱三合一主控台」
* **科幻双舱布局 (Dual-Pod Layout)**：主界面由物理分割线隔开。左舱为数据接收与插槽舱，右舱为天眼自动监控视频流。
* **物理手柄伸缩折叠**：点击分割线上的物理手柄可一键优雅将右舱【展开 / 隐藏】。在折叠状态下视频全部淡出，左舱自动过渡拉宽为 100%，以极简清爽之感呈现。
* **物理分片方格插槽 (Slots Grid)**：直观以矩阵方格排列，集齐的方格呈现翡翠绿发光，缺失的方格显示为红色框线并伴随呼吸灯频闪，指引补扫极其顺畅。
* **全通道悦耳蜂鸣反馈**：内置发声反馈。不管是 SSE 截屏、天眼监控、还是拖拽粘贴，只要捕获到一个全新分片，就会即刻发出 **`1800Hz`、`0.1s` 的悦耳短滴声**，让您闭眼即可掌握集齐状态。

### 4. 三重安全看门狗断流自愈体系
* **像素指纹帧冻结侦测**：每帧计算局部像素指纹。若连续 6 帧（约 3 秒）画面完全静止，判定视频流冻结，自动静默重启 WebRTC。
* **ZXing 超时自愈看门狗**：与 5 秒超时 Promise 竞速。若连续 3 次解析超时，自动静默重启视频轨道。
* **心跳轮询保护**：每 5 秒自动校对扫描心跳。若发现心跳停滞，立即重建连接，实现完全的“无人值守、高稳定性”传输。

## 🛠️ 环境准备与快速部署

本工具支持**完全全离线运行**，不依赖任何公网 API。

### 1. 发送端准备
* **零依赖**：发送端 `export.html` 为**纯前端单 HTML 页面**，无需安装任何额外软件或 Python 环境。
* **运行**：在任何需要发送数据的内网电脑上，直接双击使用主流浏览器打开 `export.html` 即可。

### 2. 接收端准备 (Flask 本地端 - 强烈推荐)
在需要接收数据的本地 Windows 11 设备上：
1. 确保安装了 Python (建议 3.10+)。
2. 在项目根目录下打开 PowerShell/CMD，运行以下命令快速装载依赖：
   ```bash
   pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
   ```
3. 双击运行根目录下的 [启动网页.bat](file:///c:/Users/Qiu/Desktop/program/IntranetDataTransferTool/启动网页.bat)。

> [!IMPORTANT]
> **关于管理员特权**：由于后台接收服务包含全局键盘钩子（监听全屏热键 `F8`），Windows 系统为了安全防范，可能会拦截非管理员进程的键盘监听。
> 因此，双击运行 `启动网页.bat` 时会**自动弹出 UAC 提权申请**，请选择“是（允许）”。提权后系统会强制将代码页切换为 UTF-8，彻底规避中文字符乱码，并自动在浏览器中唤起 `http://127.0.0.1:5000`。

## 📖 极速上手保姆级操作指南

### 第一步：在发送端准备并编码数据

1. 双击打开发送端 [export.html](file:///c:/Users/Qiu/Desktop/program/IntranetDataTransferTool/export.html)。
2. 选择您要传输的模式（通过顶部的高级滑动器切换）：
   * **📝 文本传输模式**：适用于传输代码段、SSL 密钥、配置文件、长篇文章。可在输入框中直接输入，或点击 **【📋 一键粘贴】** 注入。
   * **📁 文件传输模式**：适用于传输压缩包 (`.zip`)、办公文档 (`.docx`/`.pdf`)、音视频及可执行文件。可直接点击投递舱浏览文件，或直接**将本地文件拖拽入流光虚线投递舱内**。
3. **滑块容量微调 (可选)**：在主按钮上方，您可以滑动修改单块二维码的最大字符（100 - 500 字符，默认 200）。
4. 点击 **【✨ 生成分块二维码序列】**。系统将瞬间完成自适应分片，并在下方呈现一列高清二维码。
5. （可选）点击 **【🖥️ 进入极速全屏轮播放映大厅】**，让二维码序列进入自动放映状态。

---

### 第二步：在接收端接收并自愈拼合数据

双舱大融合主控台集成了三套接收通道，您可以任意选用或并行选用：

#### 🚀 通道 A：F8 极速后台截屏捕获
1. 双击运行 [启动网页.bat](file:///c:/Users/Qiu/Desktop/program/IntranetDataTransferTool/启动网页.bat) 启动本地主控台。
2. 看到发送端二维码页面（无论是网页滚动模式还是全屏自动轮播模式）。
3. 每次看到新的二维码完全展现在屏幕内，直接按一下键盘上的 **<kbd>F8</kbd>** 键！
4. 接收端会在后台静默截屏、高精度剥离并拼合分片。成功捕获新片时，网页端将响起清脆高频的 **"滴" 提示音**，表示该切片已安全进入内存桶。

#### 🎥 通道 B：天眼全自动视频流监控 (解放双手)
1. 在主控台右舱点击 **【🎥 开启屏幕全自动监控】**。
2. 在系统弹窗中，选择监控“发送端浏览器窗口”或“整个屏幕”。
3. 此时接收端右舱将挂载天眼实时监控画面。您只需让发送端进入【全屏幻灯片自动轮播】模式。
4. 天眼将以每秒 2 次的高频自动检测、捕捉、去重并亮绿左舱的插槽方格，配合声音滴声，全程完全无需人工干预，体验极佳！

#### 📥 通道 C：粘贴与拖拽导入 (备用通道)
* **粘贴板捕获**：当您在其他电脑上使用微信/截图工具截取了二维码图片，直接在接收端页面空白处按下 **<kbd>Ctrl + V</kbd>**，网页端 ZXing 引擎会自动解析并合流。
* **拖拽捕获**：支持将一整批截好的二维码图片，一次性拖拽投递入左舱蓝色的“网页备用通道”虚线框内，系统会按照文件名自动自然排序，并瞬间执行解码合流。

---

### 第三步：数据的还原下载与复制

当切片逐渐被捕获时，左舱预览区将实时呈现合流进度：
* **如果传输的是文本**：
  * 在集齐 100% 之前，文本框内会自动指出哪些分片尚在“漏单等待”中。
  * 集齐后，明文瞬间拼接无损呈现，点击右上角的 **【📋 一键复制】** 即可将完整长文本带走。
* **如果传输的是文件**：
  * 磨砂玻璃面板会实时更新进度条与方格状态。
  * 集齐后，翡翠绿物理矩阵全亮，下方将浮现 **【📥 拼图集齐！一键无损还原下载】** 的发光霓虹按钮。
  * 点击该按钮，系统将在纯离线状态下执行解码，并将文件以您**最开始上传的原始文件名**保存并下载到您的本地。

> [!TIP]
> **自愈式补扫技巧**：在传输较大文件（如 50 个切片以上）时，如果插槽矩阵里在第 12、35 位显示为红色，代表这两页在传输时被漏扫了。您无需全部重扫，只需回到发送端，将第 12、35 个二维码卡片展现到屏幕上（若在放映大厅内，可通过⏮️/⏭️按钮切过去），按一下 F8 或让天眼扫到，红色小方格会在瞬间自愈点亮为翡翠绿！
